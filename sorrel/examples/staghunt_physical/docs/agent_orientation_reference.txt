Agent Orientation Reference for ASCII Maps
============================================

This document explains how agent orientations work in the StagHunt_Physical environment.

ORIENTATION VALUES
------------------
0: NORTH (up)    - Vector: (-1, 0) - Row decreases, column unchanged
1: EAST (right)  - Vector: (0, 1)  - Row unchanged, column increases
2: SOUTH (down)  - Vector: (1, 0)  - Row increases, column unchanged
3: WEST (left)   - Vector: (0, -1) - Row unchanged, column decreases

Note: In grid coordinates, y (row) increases downward, x (column) increases rightward.

VISUAL REPRESENTATION IN ASCII MAPS
------------------------------------
In ASCII map files, agents are represented by the character 'A' (or 'P' for spawn points).
The orientation is not visually shown in the ASCII map itself - all agents appear as 'A'.
However, the orientation determines:
- Which direction the agent faces
- Which direction movement actions move the agent
- Which direction attack beams fire

MOVEMENT AND ORIENTATION
-------------------------
When simplified_movement is enabled:
- FORWARD moves in the direction the agent is facing
- BACKWARD moves opposite to facing direction
- STEP_LEFT and STEP_RIGHT are relative to facing direction

For example, if agent faces WEST (orientation 3):
- FORWARD moves left (column decreases)
- BACKWARD moves right (column increases)
- STEP_LEFT moves up (row decreases, north)
- STEP_RIGHT moves down (row increases, south)

ATTACK DIRECTION
----------------
Attack beams fire in the direction the agent is facing:
- Orientation 0 (NORTH): Attacks upward
- Orientation 1 (EAST): Attacks rightward
- Orientation 2 (SOUTH): Attacks downward
- Orientation 3 (WEST): Attacks leftward

COORDINATE SYSTEM REMINDER
---------------------------
- Rows (y): 0 is top, increases downward
- Columns (x): 0 is left, increases rightward
- Position format: (row, column) or (y, x)
